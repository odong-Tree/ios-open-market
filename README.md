### 프로젝트 규칙

 ### 브랜치 이름 규칙
- 스텝별 개인 브랜치 ex) step1-kkomal, step1-odongnamu
 
 ### 커밋 규칙
- 가능한 작은 단위로 커밋 하기 
- 실행 가능한 상태에서 커밋 하기
 
 ### 커밋 메시지 규칙
- feat: 새로운 코드나 기능 추가
- style: 기능이나 로직 변경 없이 코드 수정
- refactor: 변수 이름을 바꾸거나, 오탈자 수정
- docs: 문서 수정 및 추가
- fix: bug 수정 
- chore: 기타 잡 업무
 
 ### 코딩 컨벤션
- closer를 사용할 경우 축약형은 지양
- Swift API 디자인 가이드 라인을 준수하여 작성
- naming 신경쓰기
